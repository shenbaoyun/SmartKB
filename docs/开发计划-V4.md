# SmartKB 智能对话平台 — 开发计划 V4

> 版本：V4.0  |  日期：2026-06-23

---

## 一、项目定位

一个「**动态 Prompt 插件驱动 + 多模型自由切换 + 智能 Prompt 匹配**」的对话平台。

**核心差异化能力：**
1. **Prompt 动态插件**：系统提示词存储在 MySQL 中，运行时动态注入，改 Prompt = 改一条数据库记录，无需重启
2. **智能匹配**：用户提问时系统自动分析问题内容，从 Prompt 模板库中匹配最合适的模板，告别手动选 Prompt 的困扰
3. **多模型自由切换**：DeepSeek / 百炼通义千问一键切换，前端下拉即切

## 二、技术栈

| 层级 | 技术 | 版本 | 阿里规范依据 |
|------|------|------|-------------|
| 语言 | JDK | 21 (LTS) | 使用 LTS 版本 |
| 框架 | Spring Boot | 3.4.x | - |
| 构建 | Maven Wrapper | 3.9+ | 标准构建工具 |
| AI 编排 | LangChain4j | 1.0.0-beta5 | - |
| ORM | MyBatis-Plus | 3.5.x | 阿里推荐 MyBatis |
| SQL 方式 | XML Mapper + 注解 | - | 复杂查询走 XML |
| 数据库 | MySQL | 8.0 | - |
| 前端 | Vue 3 + Vite + Element Plus | - | - |
| 通信 | REST + SSE（流式） | - | - |

**阿里规范对照：**
- 命名：类名 `UpperCamelCase`，方法/变量 `lowerCamelCase`，常量 `CONSTANT_CASE`
- 分层：严格 `controller → service(接口) → service.impl → mapper` 四层
- PO 类：`DO` 后缀，`@TableField` 显式声明下划线映射
- 接口：Service 层面向接口编程（`XxxService` + `XxxServiceImpl`）
- 注释：所有类/方法/字段必须有 Javadoc 中文注释
- 日志：SLF4J + Lombok `@Slf4j`
- SQL：复杂查询必须拆分为多次单表查询，禁止 JOIN

│  ChatService / ConversationService            │

│  ModelConfigService                           │
└────────────────────┬─────────────────────────┘
                     │
┌────────────────────▼─────────────────────────┐
│         Mapper 层（MyBatis-Plus + XML）        │
│  ConversationMapper / MessageMapper           │

│                 MySQL 8.0                     │
└──────────────────────────────────────────────┘

### 3.2 Prompt 智能匹配系统（V4 核心）

```
用户提问 → 系统判断 promptMode
               │
    ┌──────────┴──────────┐
    │ auto（智能匹配）      │ manual（手动锁定）
    ▼                      ▼
  PromptMatcherService    直接用指定的 Prompt
    │
    ├─ 1. 关键字匹配（毫秒级，零成本）
    │     用户问题分词 → 匹配 prompt_template.match_keywords
    │     命中 → 返回模板
    │
    ├─ 2. LLM 轻量分类（~0.5s，兜底）
    │     廉价 LLM 分析意图 → 返回最佳模板 ID
    │
    └─ 3. 兜底 → "通用助手" 默认模板
           │
           └─→ 注入 SystemMessage → 主 LLM → 回答
```

### 3.3 多模型切换

```
application.yml                   前端 ModelSwitcher
┌────────────────────┐            ┌──────────────────┐
│ model:              │            │ DeepSeek V3     │
│   deepseek:         │    ───→   │ 百炼-Qwen-Max  │
│     base-url: ...   │    API    │ 百炼-Qwen-Plus │
│   bailian:          │            └──────────────────┘
│     qwen-max: ...   │
│     qwen-plus: ...  │   切换：LangChain4jConfig
└────────────────────┘        动态创建 ChatLanguageModel
```

### 3.4 对话流程（全链路）

```
1. 用户输入消息 + promptMode=auto/modelCode=qwen-max
2. ChatController.stream() 接收
3. ChatService.buildAndStream():
   a. 查询 ConversationDO（单表）
   b. 查询 PromptTemplateDO（单表，智能匹配或直接查）
   c. 查询 ModelConfigDO（单表，拿 base-url + api-key）
   d. 查询历史 MessageDO（单表，最近 20 条）
   e. 构建 ChatLanguageModel（动态 api-key + base-url）
   f. 拼接消息链：SystemMessage + 历史消息 + 当前 UserMessage
   g. 调用 LLM，SSE 流式返回
   h. 保存 USER + ASSISTANT 两条 MessageDO（单表 INSERT × 2）
   i. 更新 ConversationDO.updated_time（单表 UPDATE）
```

## 四、数据库设计

### 4.1 表结构

共 **4 张表**，全部独立单表设计，无外键约束（应用层保证一致性）。

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| conversation | 对话会话 | id, title, model_code, prompt_id, created_time, updated_time |
| message | 对话消息 | id, conversation_id, role, content, prompt_id, model_code, created_time |
| prompt_template | Prompt 模板（插件） | id, name, system_prompt, category, match_keywords, match_mode, is_preset, is_active |
| model_config | 模型配置 | id, name, code, base_url, api_key, is_active |

### 4.2 单表查询规范

```java
// 正确：三步单表查询
// 第一步：查会话
ConversationDO conversation = conversationMapper.selectById(conversationId);
// 第二步：查消息列表
List<MessageDO> messages = messageMapper.selectByConversationId(conversationId);
// 第三步：查 Prompt 名称（批量）
List<PromptTemplateDO> prompts = promptTemplateMapper.selectBatchIds(promptIds);

// 禁止：LEFT JOIN, INNER JOIN 等联表操作
```

## 五、API 设计

### 5.1 Prompt 模板管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/prompts` | 获取所有启用模板 |
| GET | `/api/prompts/{id}` | 获取单个模板 |
| **GET** | **`/api/prompts/match?message=xxx`** | **智能匹配 Prompt** |
| POST | `/api/prompts` | 新增模板 |
| PUT | `/api/prompts/{id}` | 更新模板（即时生效） |
| DELETE | `/api/prompts/{id}` | 删除（preset 不可删） |

### 5.2 模型管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/models` | 获取所有启用模型 |

### 5.3 对话管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/conversations` | 新建会话 |
| GET | `/api/conversations` | 会话列表（按更新时间倒序） |
| GET | `/api/conversations/{id}` | 会话详情（含消息列表） |
| PUT | `/api/conversations/{id}` | 更新会话 |
| DELETE | `/api/conversations/{id}` | 删除会话及消息 |
| GET | `/api/conversations/{id}/messages` | 历史消息 |

### 5.4 对话问答

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat/{conversationId}` | 非流式发送消息 |
| GET | `/api/chat/{conversationId}/stream` | **SSE 流式**发送消息 |

**ChatRequest：**
```json
{
  "content": "帮我翻译这段英文",
  "promptId": null,
  "promptMode": "auto",
  "modelCode": "deepseek-v3"
}
```
