## 一、V4 现存问题及修复方案

### 问题 1：H2 数据库持久性

| 维度 | 说明 |
|------|------|
| 当前状态 | H2 **文件模式**，数据存在 `backend/data/smartkb.mv.db` |
| 持久性 | ✅ **永久保存**，重启不丢失，除非手动删除文件 |
| 建议 | 学习和小型项目直接用 H2 文件模式；生产切换 MySQL 仅需改一行配置 |

> V5 无需改动，此处仅作说明。

### 问题 2：Prompt 管理页空白

| 根因 | 代码位置 | 修复 |
|------|---------|------|
| `PromptFormDialog.vue` 第 49 行用了 `computed()`，但只 `import { reactive, watch }`，缺少 `computed` 导入 | `frontend/src/components/prompt/PromptFormDialog.vue:40` | 补 `import { reactive, watch, computed } from 'vue'` |

### 问题 3：缺少文档上传 / RAG 能力

当前只能纯对话，无法上传文档构建知识库。V5 新增完整的 RAG 链路：

```
文档上传 → 文本分段 → Embedding 向量化 → 存入向量库
用户提问 → 向量检索 → 注入上下文 → LLM 生成回答
```

**技术方案：**

| 环节 | 技术 | 说明 |
|------|------|------|
| 文档解析 | Apache Tika（内嵌） | 支持 PDF / TXT / MD / DOCX |
| 文本分段 | LangChain4j `DocumentByParagraphSplitter` | 按段落切分 |
| Embedding | DeepSeek / 百炼 Embedding API | 复用现有模型配置 |
| 向量存储 | `InMemoryEmbeddingStore` | 内存模式，可选文件持久化 |
| 检索 | `EmbeddingStoreContentRetriever` | Top-K 语义检索 |

**新增表：**

```sql
-- 知识库文档表
knowledge_document (
    id          BIGINT AUTO_INCREMENT,
    name        VARCHAR(200)  NOT NULL,          -- 文件名
    file_type   VARCHAR(20),                     -- PDF/TXT/MD/DOCX
    file_size   BIGINT,                          -- 文件大小（字节）
    chunk_count INT          DEFAULT 0,          -- 分段数量
    status      VARCHAR(20)  DEFAULT 'PENDING',  -- PENDING → PROCESSING → READY → FAILED
    created_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**新增 API：**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/documents` | 上传文档（multipart） |
| GET | `/api/documents` | 文档列表 |
| DELETE | `/api/documents/{id}` | 删除文档及向量 |

**对话增强**：ChatService 在构建消息链时，额外调用 RAG 检索：
```
SystemMessage = Prompt系统提示词 + RAG检索到的相关文档片段
```

### 问题 4：流式输出未生效

| 根因 | 说明 |
|------|------|
| V4 的 `streamMessage` 内部调用非流式 `sendMessage`，等全部完成后一次性返回 | 前端 EventSource 接收到的是一次性文本，无打字机效果 |
| langchain4j 0.36.2 的 `StreamingChatLanguageModel` 需配合正确的 `StreamingResponseHandler` | V4 因 API 不兼容暂时回退 |

**V5 修复方案：**

```java
// 真流式调用
StreamingChatLanguageModel sm = OpenAiStreamingChatModel.builder()
        .apiKey(cfg.getApiKey()).baseUrl(cfg.getBaseUrl())
        .modelName(cfg.getModelName()).timeout(Duration.ofSeconds(120)).build();

sm.chat(msgs, new StreamingResponseHandler<AiMessage>() {
    public void onNext(String token) {
        emitter.send(SseEmitter.event().name("message").data(token));
    }
    public void onComplete(Response<AiMessage> response) {
        // 保存完整回复
        saveAssistantMessage(...);
        emitter.send(SseEmitter.event().name("done").data(...));
        emitter.complete();
    }
    public void onError(Throwable error) {
        emitter.send(SseEmitter.event().name("error").data(error.getMessage()));
        emitter.completeWithError(error);
    }
});
```

## 四、关键架构变更

### 4.1 RAG 对话链路（V5 新增）

```
用户提问
    │
    ├─→ PromptMatcherService：匹配 Prompt 模板
    │
    ├─→ RagService.search(userMessage)：
    │       ├─ 调用 EmbeddingModel.embed(userMessage)
    │       ├─ EmbeddingStore.search(embedding, maxResults=5)
    │       └─ 返回 List<TextSegment>
    │
    ├─→ 拼接 SystemMessage：
    │       = Prompt模板.systemPrompt
    │       + "以下是相关参考资料：\n"
    │       + [文档1片段] + [文档2片段] + ...
    │
    ├─→ 拼接历史消息 + 当前 UserMessage
    │
    └─→ LLM 生成回答（流式 / 非流式）
```

### 4.2 流式输出架构

```
GET /api/chat/{id}/stream?content=xxx&...
    │
    ▼
ChatController.streamMessage()
    │
    ▼
ChatService.streamMessage() → SseEmitter
    │
    ├─→ StreamingChatLanguageModel.chat(msgs, handler)
    │       │
    │       ├─ onNext(token) → emitter.send("message", token)   // 逐 token 推送
    │       ├─ onComplete(resp) → 保存消息 → emitter.send("done") → complete()
    │       └─ onError(err) → emitter.send("error") → completeWithError()
    │
    └─→ 前端 EventSource 实时渲染打字机效果
```
